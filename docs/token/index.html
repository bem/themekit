Token

Design token is an entity that allows you to capture low-level values and then use them to create styles for your product with cross-platform support, since the token is not tied to a specific language or platform.

note

A themekit supports write tokens in json or yaml format.

Aliases#

Token supports aliases for values, it's helpful for using primitives in composed tokens.

  • yaml
  • json
colorPrimary:
value: "#000"
buttonColor:
value: "{color.primary.value}"

Meta#

Token can have meta props, like comment for documentation or group for using in filter.

  • yaml
  • json
colorPrimary:
value: "#000"
comment: "Project primary color"
buttonColor:
value: "{color.primary.value}"
group: "components"

Nesting#

Token can be nested for creating hierarchical structure.

  • yaml
  • json
color:
primary:
value: "#000"
button:
color:
value: "{color.primary.value}"

Interface#

{
/**
* Token value
*/
value: string
/**
* Token comment
*/
comment?: string
}
